Rule 102.D.9).

Exceptions are a disagreement with a BRC (Billing Repair Card, as defined in Rule 83 exhibit “A”) line

exceptions may include, but are not limited to, such issues as incorrect car initials/numbers, incorrect

hours, or excessive material charges.
Y.1     How to Submit Exceptions to a Billing Party for Charges
        Received for AAR Car Repair Billing
This section lists guidelines for submitting exceptions to a Billing Party.
There are several types of exceptions to AAR Car Repair Bills:
 1.     Unpaid Invoices – follow AAR OM Rule 112.H
 2.     Paid Invoices – follow AAR OM Rule 112.H.4
Exceptions and exception communications are handled using the CRB Electronic Exceptions System and
are processed in real time. Electronic billing exceptions will be submitted through the CRB Electronic
Exceptions System, and once validated, electronically forwarded to the appropriate billing party.
Electronic exceptions are herein referred to as “Exception Letters”. Billing parties will use this same
system to respond to exception letters and either agree, partially agree, or disagree with each
exception. Billing parties will also use this system to issue CBAs (Counter Billing Authority) to a billed
party when in agreement or partial agreement on a paid invoice as defined in Y.2.
Exception letters shall be submitted at the invoice level and submitted either through an FTP (File
Transfer Protocol) mailbox or through an upload function using the CRB Electronic Exceptions Upload on
the CRB Dashboard. In either case, every inbound file will be validated before accepted in accordance
with Y.1.2. Validated messages will be stored and forwarded to the appropriate receiver. Invalid
messages will not be saved and will be rejected. In both cases, the CRB participant(s) listed to receive
exception submission notifications for the submitting party will be notified via email of the message
validation status. Participant notifications are managed in CRB under the Data Exchange tab and are the
responsibility of the participant CRB Content Admin to maintain.

Y.2     Issuing Electronic Counter Billing Authority (CBA)
This section lists guidelines for issuing electronic CBAs to a receiving party.
There are several reasons a CBA is issued:
 1.     Paid Invoice Exception Response – AAR Rule 112.D
 2.     MID Inspection – AAR Rule 120.B
 3.     AAR Circular Letter – AAR Rule 112.D
Per Rule 112.D, “Counter Billing Authority (CBA) will be used for the purpose of monetary adjustment
for the parties concerned.” CBAs will be issued using the CRB Electronic Exceptions System and
processed in real time. CBAs may be issued in response to paid invoice exception letters (Rule 112.D), as
the result of an MID Inspection (Rule 120.B), or in response to an AAR Circular Letter claim from a
collecting party (Rule 112.D).
All CBA messages shall be formatted in accordance with Y.2.5 as Format 3 messages. Both FTP and the
CRB Exceptions Dashboard can be used to submit CBA messages. CBA messages in response to a paid
exception will be included with the Type 2 messages in the response file while MID and Circular Letter
CBA messages will be submitted as independent messages. All CBA messages submitted through an FTP
mailbox shall be in a 500-byte format. Both .txt and .dat files are accepted. CBA Messages submitted
through the CRB exceptions upload function on the CRB Exceptions Dashboard shall be in .xlsx format. In
all cases, the exception message shall be in a .zip file with all applicable attachments.
When a CBA message is issued in response to a paid invoice exception via FTP in a 500-byte format, the
Format 3 CBA message should be the last record in the response, preceded by the Format 2 records.
When submitted as a .xlsx upload through the exceptions dashboard, the Format 3 CBA Message will be
submitted using the CBA CA Template tab on the “Exception Template with CBA”, available for
download through the CRB Electronic Exceptions Dashboard.
Each Record Format 3 message allows for 3 attachments per record, 1 in each attachment field. There is
no limit to the number of attachments in the .zip file however no single attachment can exceed 10 MB.
The Attachment 1 field is mandatory whereas Attachment 2 and Attachment 3 are optional. Attachment
1 must list the file name of the CBA Letter issued and a copy of the CBA Letter must be included in the
.zip file. Attachment 2 and 3 can be used for any additional attachments the submitter deems needed
for the record. Only PDF documents are accepted in the electronic exceptions system. The CBA file will
be rejected if any attachments in the .zip file are not correctly listed in the CBA message, or an
attachment is listed in the CBA message but is not included in the .zip file. File names in the exception
message and .zip file must match for validation.
Y.2.1  Issuing a Paid Invoice CBA
Exception letters submitted by a billed party to the billing party for a paid invoice, in accordance with
AAR Office Manual Rule 112.H are submitted using the CRB Electronic Exceptions System. When a paid
invoice exception letter is received, the billing party will respond to each exception letter with a
Response Indicator of A (Agree), P (Partially Agree) or D (Disagree). Response Indicators A and P will be
accompanied by a Detail Source EZ, indicating a CBA issued for that specific exception. Additionally, an
Amount Indicator of C (Credit) or D (Debit) will accompany a Detail Source EZ to indicate if the amount
for that exception item is a debit or credit to the receiving party.

Car Repair Billing Procedures Manual                                                                 217

When an exception letter response message contains at least 1 Detail Source EZ, a singular electronic
CBA with Detail Source CA, must be included in the response. The electronic CBA is in response to that
specific exception letter and does not preclude the billed party from submitting additional exception
letters in the future for unresolved exceptions. The electronic CBA will be included as a Format 3 Record
within the exception response message and should be the last record in the message. All electronic CBA
messages must be formatted in accordance with Y.2.4.
Y.2.2 Issuing an MID Inspection CBA
In accordance with AAR Rule 120.B, “Whenever the AAR finds that a specific AAR Interchange Rule(s) has
been violated, the Executive Director must direct the payment of refunds or adjustments of
responsibility.” When this occurs, an electronic CBA will be submitted by the responsible party using
Detail Source MI, indicating a CBA issued as a result of an MID Inspection. MID CBAs will be standalone
CBA messages and require no request or exception from the receiving party. Additionally, MID CBA
messages will not support responses to the CBA in the CRB Electronic Exception System.
Y.2.3 Issuing a Circular Letter CBA
In accordance with AAR Rule 112.D, an AAR Circular Letter may be issued that entitles an affected party
to a collection of monies. Once the affected party has satisfied their responsibility in Rule 112.D.2, an
electronic CBA will be submitted by the responsible party using Detail Source CL, indicating a CBA issued
as a result of an AAR Circular Letter claim. CL CBAs will be standalone CBA messages and will not support
responses to the CBA in the CRB Electronic Exception System.
Y.2.4 MID and Circular Letter CBA Naming Convention
Because a CBA with Detail Source CA is included in the exception letter response message, no specific
naming convention is required. However, for CBAs issued with Detail Source MI or CL, a specific file
naming convention is needed to assist in identifying when a MID or Circular Letter CBA is submitted and
display the file correctly in the dashboard. The CBA message and the .zip container will both use the
same naming convention as follows:
For MID CBAs:
SubmitterMark_ReceiverMark_MI_CBAReferenceNumber_YYMM.zip
For CL CBAs:
SubmitterMark_ReceiverMark_CL_CBAReferenceNumber_YYMM.zip
